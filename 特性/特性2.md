核心区别
Handle(pattern string, handler Handler)
接收已经实现了 http.Handler 接口的对象。
HandleFunc(pattern string, handler func(ResponseWriter, *Request))
接收普通函数，内部自动把函数转为 HandlerFunc 类型，而 HandlerFunc 本身实现了 Handler 接口。
2. 拆解源码关键代码
go
运行
// Handle 直接接收 Handler 接口实现
func Handle(pattern string, handler Handler) {
    DefaultServeMux.register(pattern, handler)
}
go
运行
// HandleFunc 接收一个普通函数，做了类型转换
func HandleFunc(pattern string, handler func(ResponseWriter, *Request)) {
    // HandlerFunc(handler) 把普通函数包装成 Handler 对象
    DefaultServeMux.register(pattern, HandlerFunc(handler))
}
类型定义：
go
运行
type HandlerFunc func(ResponseWriter, *Request)

// HandlerFunc 自动实现了 Handler 接口
func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
    f(w, r)
}
3. 使用形式不一样
写法 1：用 Handle（传结构体实例）
go
运行
type H struct{}
func (H) ServeHTTP(w http.ResponseWriter, r *http.Request) {}

http.Handle("/", H{})
写法 2：用 HandleFunc（直接传函数）
go
运行
func fn(w http.ResponseWriter, r *http.Request) {}

http.HandleFunc("/", fn)
等价写法（手动转换）：
go
运行
http.Handle("/", http.HandlerFunc(fn))
4. 本质结论
Handle：接收已经实现接口的类型实例。
HandleFunc：接收普通函数，内部自动包装成 HandlerFunc，再调用 Handle 逻辑。
最终进入路由注册表的，一定都是 Handler 接口实现。
5. 开发怎么选
简单路由、只写一个处理函数：优先用 HandleFunc，代码最简。
中间件、自定义 Handler 结构体、需要携带成员变量：用 Handle 传入自定义结构体。
