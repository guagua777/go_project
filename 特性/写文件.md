2. go写入文件
```
package main

import (
	"os"
)

func main() {
	content := []byte("hello go file\n")
	// 覆盖写入，不存在则创建
	err := os.WriteFile("test.txt", content, 0644)
	if err != nil {
		panic(err)
	}
}


// 第一个参数为文件路径
// 第二个参数为写入内容
// 第三个参数为文件权限
// 其中写入内容
os.WriteFile("test.txt", content, 0644)

其中的内容参数为切片
3. 为什么设计成切片？
数组长度是类型的一部分，[10]byte 和 [20]byte 是两种不同类型，函数没法接收任意长度数据；
切片动态可变长度，适合用来承载任意长度文件内容。
一句话总结：
参数类型是 []byte = 切片；即使你传入数组变量，编译器也会隐式转为切片。


content := []byte("hello go file\n")
也是切片
[]byte{...} 字面量：创建切片

[10]byte{...}：// 切片
b1 := []byte("abc")
// 数组（定长）
var b2 [4]byte = [4]byte{'a','b','c','\n'}才是数组


字符串强转 []byte，返回值永远是字节切片，不会是数组。
一句话记忆：
方括号里没有数字 = 切片；
方括号里有固定数字 = 数组。


panic 程序终止
与 recover 恢复程序执行
```


3. go的http处理
```
package main

import (
	"net/http"
)

func main() {
	// 注册路由 + 处理器
	http.HandleFunc("/hello", func(w http.ResponseWriter, r *http.Request) {
		w.Write([]byte("Hello HTTP"))
	})

	// 启动服务 :8080
	_ = http.ListenAndServe(":8080", nil)
}


// 方式1
func handler(w http.ResponseWriter, r *http.Request) {
	w.WriteHeader(200)
	_, _ = w.Write([]byte("ok"))
}

http.HandleFunc("/", handler)

// 方式2， 实现http.Handler 接口
为什么实现这个接口，就可以作为handler了呢
type Handler interface {
    ServeHTTP(ResponseWriter, *Request)
}
只要某个类型拥有 ServeHTTP 方法，就自动实现了 http.Handler 接口

注册路由的函数
func Handle(pattern string, handler Handler)
第二个参数要求传入 http.Handler 接口类型

// fangshi 3
// The HandlerFunc type is an adapter to allow the use of
// ordinary functions as HTTP handlers. If f is a function
// with the appropriate signature, HandlerFunc(f) is a
// [Handler] that calls f.
type HandlerFunc func(ResponseWriter, *Request)

// ServeHTTP calls f(w, r).
func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
	f(w, r)
}
```
