第100行的 HandlerFunc(handler) 是强转吗

是的，HandlerFunc(handler) 也是类型转换（强转），和上一题的 fileLog(destination) 完全一样的原理。

关键定义
在 server.go:2216 中：


go
type HandlerFunc func(ResponseWriter, *Request)
HandlerFunc 的底层类型是 func(ResponseWriter, *Request)，而 handler 参数就是一个普通的函数，签名也是 func(ResponseWriter, *Request)。所以 HandlerFunc(handler) 就是把普通函数转为命名类型 HandlerFunc。

为什么要转？
因为 HandlerFunc 实现了 Handler 接口。看 server.go:2219-2221：


go
func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
	f(w, r)
}
转换链路是这样的：


plainText
普通函数 func(ResponseWriter, *Request)
        ↓  类型转换 HandlerFunc(handler)
命名类型 HandlerFunc（拥有 ServeHTTP 方法）
        ↓  隐式满足接口
Handler 接口（要求 ServeHTTP 方法）
普通的 func(ResponseWriter, *Request) 没有 ServeHTTP 方法，不能赋给 Handler 接口。但转成 HandlerFunc 之后，它就"自动拥有"了 ServeHTTP 方法，可以当作 Handler 使用。

这是 Go 的经典惯用法
HandlerFunc 本质上是一个适配器（adapter），让普通函数能适配接口。这也是 http.HandleFunc 能直接传入普通函数的底层原理：


go
// 用户写法：传入普通函数
http.HandleFunc("/log", func(w http.ResponseWriter, r *http.Request) { ... })

// 内部：强转为 HandlerFunc，再作为 Handler 存储
mux.handle(pattern, HandlerFunc(handler))
