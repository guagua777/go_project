如果你改成指针接收者：
go
运行
func (*MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request)
那么下面这行会编译报错：
go
运行
http.Handle("/", MyHandler{}) // 值实例，不再实现Handler接口
必须传指针才能正常运行：
go
运行
http.Handle("/", &MyHandler{})
二、核心原理（Go 接口实现规则）
设类型 T：
值方法：func (T) ServeHTTP ()
值类型 T{} 实现接口
指针类型 &T{} 也自动实现接口
*指针方法：func (T) ServeHTTP()
只有指针 &T{} 实现接口
值 T{} 不实现接口
原因：
Go 只会自动把值转为指针，不会把指针自动转为值。
值对象拿不出指针方法。
验证代码：
go
运行
// 指针方法
func (*MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {}

// 报错：MyHandler{} 没有实现 http.Handler
var _ http.Handler = MyHandler{}

// 正常：&MyHandler{} 实现了接口
var _ http.Handler = &MyHandler{}
三、什么时候用值接收者，什么时候用指针接收者？
1. 使用值接收者 (MyHandler)
适用条件：
方法内部只读结构体，不修改任何字段
结构体很小，拷贝成本可以忽略
优点：
传值 MyHandler{}、传指针 &MyHandler{} 都能满足接口，写法更灵活。
go
运行
func (MyHandler) ServeHTTP(...) {}
2. 使用指针接收者 (*MyHandler)
必须满足下面任意一条：
方法内部要修改结构体成员变量；
结构体很大，不想每次调用都复制一整个对象，节省内存；
希望全局只共用同一个实例，避免多份副本。
一旦写成指针方法：
只能传入 &MyHandler{}，值实例不再满足接口。
go
运行
type MyHandler struct {
	count int
}

// 修改成员，必须用指针接收者
func (h *MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	h.count++
}
四、总结口诀
值方法：值和指针都能实现接口
指针方法：只有指针能实现接口
不改字段、小结构体 → 值接收者
改字段、大结构体、单例复用 → 指针接收者
