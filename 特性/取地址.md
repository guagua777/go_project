& 是 Go 中的取地址运算符，用来获取变量的指针（内存地址）。

具体到这行代码

go
mh := myHandler{}       // mh 是 myHandler 类型的值
Handler: &mh,           // &mh 是 *myHandler 类型（指向 mh 的指针）
为什么要用 &mh 而不是 mh？
因为 http.Server 的 Handler 字段类型是 http.Handler 接口：


go
type Handler interface {
    ServeHTTP(ResponseWriter, *Request)
}
而你的 ServeHTTP 方法的接收者是指针类型：


go
func (m *myHandler) ServeHTTP(w http.ResponseWriter, r *http.Request)
这意味着 只有 *myHandler（指针类型）实现了 Handler 接口，myHandler（值类型）没有实现。

类型	是否实现了 Handler 接口	原因
myHandler	❌	值类型没有 ServeHTTP 方法
*myHandler	✅	指针类型有 ServeHTTP 方法
所以必须传 &mh（指针），如果传 mh（值），编译器会报错：myHandler does not implement Handler。

& 的本质

plainText
mh      →  myHandler{}   （值，存在内存某处）
&mh     →  0xc000xxxxx   （该值的内存地址/指针）
&mh 就是"给我 mh 这个变量的地址"，得到一个 *myHandler 类型的指针，这个指针满足 http.Handler 接口，所以能赋给 Handler 字段。
